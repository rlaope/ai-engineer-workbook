# Volume 29 — 임베딩 평가와 벤치마크

> 이 권이 끝나면 *우리 도메인에 어떤 임베딩 모델을 쓸지*를 30분 안에 결정할 수 있게 됩니다.

## 목적

임베딩 모델은 모델 카드만으로 비교할 수 없습니다. 같은 모델이 영어에서는 강하지만 한국어에서는 약할 수 있고, 일반 도메인에서는 강하지만 의료·법률에서는 약할 수 있습니다. MTEB·BEIR·KoMTEB 같은 벤치마크는 이 *도메인별·작업별* 강약을 드러냅니다. 이 권은 평가 벤치마크의 사고법과 자체 평가 셋 구성을 다룹니다.

## 선수 지식

- Volume 38, 34, 35 완료
- 외부 지식: 검색 평가 지표

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. MTEB 의 작업 분류(분류·군집·검색·STS 등) 를 알 수 있습니다.
2. BEIR 가 *Zero-Shot 검색 능력*을 측정함을 이해합니다.
3. nDCG·Recall@k·MRR·MAP 의 차이를 손으로 계산할 수 있습니다.
4. 자체 도메인 평가 셋을 100~500 쌍으로 구축할 수 있습니다.
5. 임베딩 모델 선택의 5 가지 기준(언어·도메인·차원·속도·라이선스) 을 갖습니다.

## 챕터 목차

1. **왜 자체 평가 셋이 필요한가**
2. **MTEB** — Massive Text Embedding Benchmark

4. **KoMTEB·CMTEB** — 한국어·중국어 벤치마크
5. **검색 평가 지표** — nDCG·Recall@k·MRR·MAP
6. **자체 평가 셋 구축 워크플로**
7. **임베딩 모델 선택 5 기준**
8. **벤치마크 함정** — 오염·과적합
