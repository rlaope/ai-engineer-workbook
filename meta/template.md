# 권 본문 표준 템플릿

이 문서는 모든 권의 본문이 따라야 하는 *고정 골격* 입니다. 새 권을 시작할 때 이 템플릿을 복사해 슬롯을 채웁니다.

---

## 권 골격

```markdown
# Volume {N} — {권 제목}

> {한 줄 목적 — "이 권이 끝나면 ~ 할 수 있게 됩니다"}

## 목적

{3-5 문장. 이 권이 왜 필요한지, 다음 권으로 가기 전 이 단계를 거쳐야 하는 이유.}

## 선수 지식

- Volume {a}, {b} 완료
- 외부 지식: {예: 고등학교 수학 미분 / Python 기본 문법}

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. ...
2. ...
... (5-8 항목)

---

## 이 권을 읽기 전에

{권 도입 — 4-6 단락. 다음 요소를 포함:
 - 이 분야가 왜 중요한가의 큰 그림
 - 이 권이 다른 권들과 어떻게 연결되는가
 - 코드 비중·수식 비중·읽는 시간 가이드
 - 학습 노트 작성 권유}

---

## 1. {챕터 1 제목}

### 1.1 {소제목 — 정의·역사·맥락}

{단락 1 — 도입}
{단락 2 — 개념 설명}
{단락 3 — 역사적 맥락 또는 흔한 오해}

### 1.2 {소제목 — 핵심 원리}

{본문 + 수식 (LaTeX) + ASCII 다이어그램}

### 1.3 {소제목 — 코드 또는 실전}

```python
# 실행 가능한 코드 (50-150 라인)
```

{코드 설명 + 결과 묘사}

### 1.4 {소제목 — 함정·트러블슈팅}

{이 영역에서 자주 마주치는 5 가지 함정과 대응}

### 1.5 챕터 정리

{한 단락 — 무엇을 배웠고 다음 챕터로 어떻게 이어지는가}

---

## 2-N. (위와 같은 구조 반복)

각 권 8-12 챕터.

---

## 권 정리

이 권에서 우리는 다음을 배웠습니다.

- {핵심 1}
- {핵심 2}
... (8-12 항목)

가장 기억에 남겨야 할 한 줄은 **"{이 권의 한 줄 요약}"** 입니다.

다음 권은 [Volume {N+1} — {제목}]({링크}) 입니다. 거기서는 {다음 권 핵심} 을 다룹니다.

---

## 추가 읽을거리

- {책·논문·공식 문서·강의} (3-5 개)

---

## 자가점검 키워드

`{키워드 1}`, `{키워드 2}`, ... (8-12 개)

## 자가점검 질문

다음 질문에 막힘없이 답할 수 있을 때 다음 권으로 넘어가십시오.

1. ...
... (7-10 항목)

## 다음 권

[Volume {N+1} — {제목}]({링크})
```

---

## 챕터 본문 작성 슬롯

각 챕터는 다음 6 가지 슬롯을 *부분적으로 또는 전부* 채웁니다.

| 슬롯 | 목적 | 분량 |
|------|------|------|
| **도입 단락** | 왜 이 챕터인가 | 200-400 자 |
| **정의·역사·맥락** | 새 용어의 정의·등장 배경 | 1-2 단락 |
| **핵심 원리** | 수식·메커니즘·이유 | 2-4 단락 |
| **코드/시각자료** | 직접 검증 가능한 예시 | 50-150 줄 |
| **함정·트러블슈팅** | 자주 마주치는 5 가지 | 1-2 단락 |
| **챕터 정리** | 핵심 한 단락 | 100-200 자 |

총 챕터당 *200-400 라인* 목표. 권 전체로 *1,800-2,500 라인*.

---

## 분량 규칙

- 권 전체: **1,800-2,500 라인** (출판 단행본 한 챕터 분량)
- 챕터당: **200-400 라인**
- 코드 블록: 권당 최소 3 개 (수학·이론 권은 NumPy 검증 코드 포함)
- 시각자료 (ASCII): 권당 최소 2 개
- *분량 부풀리기 금지* — 같은 내용 반복·여러 표현으로 늘리기 금지

---

## 체크리스트 (권 작성 후)

- [ ] 격식체 "~입니다" 일관 사용
- [ ] 모호 표현 0건 (`grep -E "(보통은|일반적으로|대체로)"`)
- [ ] 금칙어 0건 (`grep -E "(과제|채용|chal|challenge|recruitment|Co-Authored-By|개월수)"`)
- [ ] 모든 코드 블록의 *입출력이 본문에서 명시*
- [ ] 외부 이미지 링크 0건
- [ ] mermaid 0건
- [ ] 권 본문 분량 1,800-2,500 라인
- [ ] 자가점검 질문 7-10 개
- [ ] `[VERIFY]` 태그 모두 처리
- [ ] 다음 권 링크가 실제 존재 파일을 가리킴
