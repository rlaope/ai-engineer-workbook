# Volume 28 — 객체 탐지와 세그멘테이션

> 이 권이 끝나면 *분류 ≠ 탐지 ≠ 세그멘테이션* 의 차이를 입력·출력 형태 관점에서 정확히 구분할 수 있게 됩니다.

## 목적

산업 현장의 비전 문제는 *이미지 한 장의 분류*보다 *어디에 무엇이 있는가* 를 묻는 경우가 많습니다. 객체 탐지는 박스로, 세그멘테이션은 픽셀 단위 마스크로 이 질문에 답합니다. 이 권은 두 작업의 정의·평가 지표(IoU·mAP)·대표 모델 계보(YOLO·Faster R-CNN·Mask R-CNN·SAM) 을 정리합니다.

## 선수 지식

- Volume 26, 27 완료
- 외부 지식: 좌표·박스 표현

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. 분류·탐지·인스턴스/시맨틱 세그멘테이션의 출력 차이를 설명할 수 있습니다.
2. IoU 와 mAP 의 정의를 손으로 계산할 수 있습니다.
3. One-Stage(YOLO) 와 Two-Stage(R-CNN 계열) 의 차이를 설명할 수 있습니다.
4. SAM(Segment Anything) 의 *프롬프트 가능 세그멘테이션* 의 의미를 알 수 있습니다.
5. 비전 모델의 데이터 어노테이션 비용 구조를 인식하게 됩니다.

## 챕터 목차

1. **세 가지 비전 작업의 정의** — 분류·탐지·세그멘테이션
2. **IoU 와 mAP** — 평가 지표의 의미
3. **앵커 박스** — 후보 영역의 격자
4. **R-CNN → Fast R-CNN → Faster R-CNN** — 두 단계 탐지의 진화
5. **YOLO 계열** — 한 단계로 탐지
6. **시맨틱 세그멘테이션** — U-Net·DeepLab
7. **인스턴스 세그멘테이션** — Mask R-CNN
8. **Segment Anything (SAM)** — 프롬프트 가능 분할

## 자가점검 키워드

`IoU`, `mAP`, `앵커 박스`, `R-CNN`, `YOLO`, `U-Net`, `Mask R-CNN`, `SAM`
