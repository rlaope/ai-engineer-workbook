# Volume 75 — Agent Memory 시스템

> 이 권이 끝나면 *에이전트가 어제 말한 것을 오늘 기억하게 만드는* 시스템을 직접 설계할 수 있게 됩니다.

## 목적

LLM 자체는 컨텍스트 윈도우만큼만 기억합니다. 진짜 유용한 에이전트는 *세션을 넘어선 기억*을 가져야 하며, 이 기억은 단기·장기·에피소드·시맨틱·절차적 기억으로 분류됩니다. Mem0·Letta·LangGraph Memory 같은 라이브러리는 이 추상화를 제공합니다. 이 권은 에이전트 메모리 시스템 설계를 다룹니다.

## 선수 지식

- Volume 35, 41 완료
- 외부 지식: 데이터베이스의 일반 개념

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. 단기·장기·에피소드·시맨틱·절차적 기억의 차이를 그릴 수 있습니다.
2. 메모리 *쓰기* 와 *조회* 의 두 단계 설계를 합니다.
3. Mem0·Letta 의 차이를 알 수 있습니다.
4. 메모리 압축·요약·삭제 정책을 설계할 수 있습니다.
5. 메모리의 *프라이버시 함정* 을 인식합니다.

## 챕터 목차

1. **기억의 분류** — 인지심리학에서 차용
2. **단기 vs 장기 기억의 시스템 분리**
3. **에피소드·시맨틱 메모리** — 이벤트 vs 사실
4. **절차적 메모리** — 학습된 워크플로
5. **Mem0** — 자동 추출·갱신
6. **Letta (formerly MemGPT)** — Self-Editing Memory
7. **메모리 압축·요약·삭제**
8. **프라이버시와 보존 정책**
