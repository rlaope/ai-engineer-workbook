# Volume 40 — RAG — 검색 증강 생성

> 이 권이 끝나면 빈 회사 문서 더미를 받았을 때 *답변 가능한 챗봇*을 만드는 표준 파이프라인을 그릴 수 있게 됩니다.

## 목적

LLM 의 가장 큰 약점은 *학습 시점 이후의 지식과 사내 비공개 데이터를 모른다는 것*입니다. RAG 는 모델이 답을 만들기 전에 *관련 문서를 검색해 컨텍스트로 주입*함으로써 이 약점을 보완합니다. 백엔드 엔지니어가 가장 자주 만나게 될 LLM 응용이며, 시스템 설계 관점에서 가장 정형화된 영역입니다. 이 권은 RAG 의 표준 파이프라인과 함정을 정리합니다.

## 선수 지식

- Volume 34, 35, 39 완료
- 외부 지식: 검색 엔진의 일반 개념

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. RAG 의 5단계 파이프라인(인덱싱·검색·재순위·컨텍스트 합성·생성) 을 그릴 수 있습니다.
2. 청킹 전략의 트레이드오프를 설명할 수 있습니다.
3. Hybrid Search 와 Cross-Encoder 재순위의 동기를 알 수 있습니다.
4. RAG 평가 지표(Recall·MRR·Faithfulness·Answer Relevance) 를 구분할 수 있습니다.
5. 환각·인용 누락·시간성·다국어 같은 RAG 의 함정을 인식합니다.

## 챕터 목차

1. **RAG 의 동기** — 학습되지 않은 지식에 답하기
2. **인덱싱 단계** — 로더·청킹·임베딩
3. **청킹 전략** — 고정·문장·재귀·시멘틱
4. **검색 단계** — 벡터·키워드·하이브리드
5. **재순위** — Cross-Encoder·LLM Re-ranker
6. **컨텍스트 합성** — Context Window 설계
7. **생성 단계** — 프롬프트 템플릿·인용 강제
8. **RAG 평가** — Ragas·TruLens
9. **함정과 안전 장치** — 환각·시간성·인용

## 자가점검 키워드

`인덱싱`, `청킹`, `Hybrid Search`, `재순위`, `Cross-Encoder`, `Ragas`, `Faithfulness`, `Context Window`
