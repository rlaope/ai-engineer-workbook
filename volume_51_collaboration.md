# Volume 51 — 협업과 의사소통

> 이 권이 끝나면 PM·연구자·SRE·데이터팀과의 인터페이스에서 *어떤 정보를 어떤 형식으로 주고받아야 하는지*를 답할 수 있게 됩니다.

## 목적

AI 엔지니어의 결과물은 혼자 만들어지지 않습니다. 제품 매니저는 *비즈니스 가치*를, 연구자는 *모델 정확도*를, SRE 는 *지연/가용성*을, 데이터팀은 *데이터 품질*을 우선시합니다. 같은 모델을 두고도 우선순위가 다르므로, 의사소통 실패는 잘못된 모델을 잘못된 환경에 배포하는 결과로 이어집니다. 이 권은 직무 간 인터페이스의 표준 산출물과 회의 패턴을 정리합니다.

## 선수 지식

- Volume 1, 2 완료
- 외부 지식: 기본적인 협업 도구(Slack·Notion·Jira) 사용

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. 모델 카드·실험 보고서·롤아웃 계획서를 작성할 수 있습니다.
2. PM 과 OKR/KPI 를 모델 메트릭으로 변환할 수 있습니다.
3. 연구자의 실험 결과를 프로덕션 의사결정으로 연결할 수 있습니다.
4. 인시던트 발생 시 SRE 와의 표준 협업 흐름을 진행할 수 있습니다.
5. 데이터팀과 데이터 계약(스키마·SLA·품질 지표) 을 협의할 수 있습니다.

## 챕터 목차

1. **인접 직무의 우선순위 지도** — 누가 무엇을 가장 신경쓰는가
2. **모델 카드와 실험 보고서** — 결과를 전달하는 표준 산출물
3. **OKR/KPI → 모델 메트릭 변환**
4. **롤아웃 계획서** — 단계·중단 기준·롤백
5. **연구자와의 협업** — 실험 → 프로덕션 다리
6. **SRE 와의 협업** — 온콜·장애 회피·런북
7. **데이터팀과의 협업** — 데이터 계약·품질 SLA
8. **회의 패턴과 비동기 의사소통**
