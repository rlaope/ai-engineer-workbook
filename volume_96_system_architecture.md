# Volume 96 — AI 시스템 아키텍처 종합

> 이 권이 끝나면 *AI 애플리케이션의 표준 5 계층 아키텍처* 를 종이 위에 그릴 수 있게 됩니다.

## 목적

실무 LLM 애플리케이션은 *모델 호출* 한 줄이 아닙니다. *컨텍스트 보강 → 가드레일 → 라우터·게이트웨이 → 캐시 → 에이전트* 로 이어지는 5 계층 위에 *모니터링·오케스트레이션·피드백* 이 함께 돕니다.

## 선수 지식

- Volume 66, 70, 72, 86, 92 완료

## 학습 결과

1. AI 애플리케이션 5 계층을 그림으로 그릴 수 있습니다.
2. 모델 라우터·게이트웨이의 동기를 알 수 있습니다.
3. 시멘틱 캐시·프롬프트 캐시·응답 캐시의 적용 시점을 구분합니다.
4. 에이전트 패턴이 어디에 들어가는지 위치 잡을 수 있습니다.
5. 관측성·오케스트레이션 도구를 한 그림에 위치시킬 수 있습니다.

## 2. 계층 1 — 컨텍스트 보강

### 2.1 도구 통합

```python
def enrich_context(query, user_id):
    rag_results = rag.search(query, k=5)
    memory = memory_store.get(user_id)
    tools_available = get_tools_for_user(user_id)
    
    return {
        "rag": rag_results,
        "memory": memory,
        "tools": tools_available,
    }
```

이 단계가 *답변 품질의 가장 큰 결정자*.

## 4. 계층 3 — 라우터·게이트웨이

### 4.1 라우팅 로직

```python
def route(query):
    intent = classifier.predict(query)
    if intent == "simple_qa":
        return small_model
    elif intent == "code":
        return code_model
    elif intent == "complex_reasoning":
        return large_model
```

### 4.2 게이트웨이 도구

- **LiteLLM** — 100+ 모델 통합 인터페이스
- **Portkey** — 라우팅·캐시·관측성

## 5. 계층 4 — 캐시

### 5.1 종류

- *응답 캐시* — 정확한 프롬프트 매칭
- *시멘틱 캐시* — 의미적 유사 매칭
- *프롬프트 캐시* — 시스템 프롬프트 KV 재사용
- *임베딩 캐시* — 동일 텍스트 임베딩

### 5.2 효과

대규모 트래픽에서 *비용 50%+ 절감* 가능.

## 7. 모니터링·오케스트레이션

### 7.1 관측성

### 7.2 오케스트레이션

## 8. 참조 아키텍처

대표 사례:

- **ChatGPT-like Chatbot** — 5 계층 + 메모리
- **Coding Agent (Claude Code)** — 5 계층 + 강한 도구
- **Customer Support** — 5 계층 + 인간 핸드오프
- **Internal Q&A** — 5 계층 + 사내 RAG

각 사례가 5 계층의 *변형*.

## 권 정리

- 5 계층 = 컨텍스트·가드레일·라우터·캐시·모델
- 라우터 = 의도별 모델 선택
- 캐시 = 비용 절감의 가장 큰 도구
- 관측성 (Helicone·Langfuse) = 디버깅·개선 기반
- 오케스트레이션 = 복잡한 워크플로

가장 기억할 한 줄: **"AI 애플리케이션은 모델 한 줄이 아닌 5 계층 시스템이며, 각 계층의 책임이 명확할 때 시스템이 확장 가능해진다."**
