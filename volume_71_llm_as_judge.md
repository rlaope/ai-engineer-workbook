# Volume 60 — LLM-as-Judge 평가 자동화

> 이 권이 끝나면 *모델이 모델을 평가한다* 는 패턴의 강점과 함정을 모두 알게 됩니다.

## 목적

생성 모델 평가는 *정답이 하나가 아닌* 경우가 대부분이라 자동 평가가 어렵습니다. LLM-as-Judge 는 평가 자체를 LLM 에게 맡겨 자동화·확장하는 패턴이며, MT-Bench·Arena·Ragas 등 대부분의 현대 평가 프레임워크가 이 위에서 동작합니다. 그러나 LLM 평가자는 *위치 편향·길이 편향·자기 편애*를 가지므로, 이를 인식하고 보완하는 설계가 필수입니다.

## 선수 지식

- Volume 38, 76 완료
- 외부 지식: 인간 평가의 일반 절차

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. LLM-as-Judge 의 3 가지 평가 양식(Pointwise·Pairwise·Listwise) 을 구분할 수 있습니다.
2. 평가용 프롬프트를 *기준·예시·형식 강제* 와 함께 설계할 수 있습니다.
3. 위치·길이·자기 편애의 함정을 식별하고 완화할 수 있습니다.
4. Arena·ELO 기반 비교 평가 시스템의 동작을 알 수 있습니다.
5. 인간 평가와 LLM 평가의 정렬 측정(상관계수·Cohen's κ) 방법을 압니다.

## 챕터 목차

1. **자동 평가의 한계와 LLM-as-Judge 의 동기**
2. **세 가지 양식** — Pointwise·Pairwise·Listwise
3. **평가용 프롬프트 설계**
4. **편향 — 위치·길이·자기 편애·말투**
5. **편향 완화** — 순서 무작위·길이 보정·다중 평가자
6. **Arena·ELO 시스템** — Chatbot Arena 모델
7. **평가자 모델 선택 — 비용·정확도**
8. **인간 평가와의 정렬 측정**

## 자가점검 키워드

`Pointwise`, `Pairwise`, `Listwise`, `위치 편향`, `자기 편애`, `Arena`, `ELO`, `Cohen's κ`
