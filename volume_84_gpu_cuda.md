## 1. GPU vs CPU

### 1.1 설계 철학

- **CPU** — *단일 스레드 지연 최소화*. 큰 캐시·복잡한 분기 예측·OoO 실행. 4-64 코어.
- **GPU** — *처리량 최대화*. 작은 코어 *수만 개*. 단순한 분기·작은 캐시.

### 1.2 적용 영역

- CPU — *순차적·분기 많은* 작업 (OS·DB·웹서버)
- GPU — *대량 병렬·동일 연산* 작업 (행렬 곱·이미지 처리)

신경망의 *행렬 곱* 이 GPU 에 완벽히 맞음.

---

## 2. GPU 의 실행 모델

### 2.1 계층 구조

```
SM (Streaming Multiprocessor) — H100 의 132 개
  └── Warp — 32 개 스레드 묶음 (SIMT 실행)
        └── Thread — 가장 작은 실행 단위
```

### 2.2 워프

GPU 의 *기본 실행 단위*. 32 개 스레드가 *같은 명령을 동시* 실행 (SIMT — Single Instruction Multiple Thread).

워프 내 *분기* 가 발생하면 *직렬 실행* → 성능 손실.

---

## 3. 메모리 계층

```
+------------------+-------+----------+
| 메모리           | 크기  | 대역폭   |
+------------------+-------+----------+
| Register         | KB    | TB/s     |
| Shared Memory    | ~228KB/SM | TB/s |
| L2 Cache         | 50MB  | TB/s     |
| HBM (Global)     | 80GB  | 3TB/s    |
| Host RAM (PCIe)  | TB    | 32GB/s   |
+------------------+-------+----------+
```

H100 기준 추정. 위로 갈수록 *작고 빠름*.

---

## 4. Tensor Core

NVIDIA 의 *행렬 곱 전용 하드웨어*. Volta 부터 도입.

각 Tensor Core 가 *작은 행렬 곱 (예: 4×4×4)* 을 *한 사이클* 에 처리. 일반 코어보다 *수십 배 빠름*.

H100 의 4 세대 Tensor Core 는 *FP8 도 지원* — 더 큰 처리량.

---

## 5. 자료형

### 5.1 비교

```
+--------+------+------+------+--------+--------+
| 자료형 | 부호 | 지수 | 가수 | 범위   | 정밀도 |
+--------+------+------+------+--------+--------+
| FP32   |  1   |  8   |  23  | ±10^38 | 7 자리 |
| FP16   |  1   |  5   |  10  | ±65K   | 3 자리 |
| BF16   |  1   |  8   |  7   | ±10^38 | 2-3 자리|
| FP8    |  1   |  4   |  3   | ±240   | 1 자리 |
| INT8   |  1   |  -   |  7   | ±128   | 정수    |
+--------+------+------+------+--------+--------+
```

### 5.2 학습 vs 추론

- 학습: BF16 (FP16 의 표현 범위 부족 회피)
- 추론: FP16, INT8, FP8 (속도 우선)

---

## 6. 모델 메모리 계산

### 6.1 가중치

```
파라미터 수 × 자료형 바이트
70B × 2 bytes (FP16) = 140 GB
70B × 1 byte (INT8)  = 70 GB
70B × 0.5 bytes (FP4) = 35 GB
```

### 6.2 추론 메모리

```
가중치 + KV 캐시 + 활성화

KV 캐시 = 2 × 층 수 × hidden_dim × seq_len × batch × 2 bytes

LLaMA 3 70B, seq=2048, batch=1 추정:
가중치: 140 GB
KV 캐시: 2 × 80 × 8192 × 2048 × 1 × 2 = ~5 GB
합계: ~145 GB → H100 80GB 두 장 필요
```

### 6.3 학습 메모리

```
가중치 + 그래디언트 + 옵티마이저 상태 + 활성화
= 가중치 × 약 4-8 배

70B FP16 학습 = 약 700 GB+ → H100 8 장 (FSDP)
```

이 계산을 *손으로 할 수 있는 것* 이 핵심 역량.

---

## 7. nvidia-smi 읽기

```
+-----------------------------------------------------------------------------+
| GPU-Util  Compute M.|     Memory-Usage     |  Power Cap |  Temperature      |
| 75%      Default    |  60GB / 80GB         |  300W/350W |  65°C             |
+-----------------------------------------------------------------------------+
```

각 숫자의 의미:
- *GPU-Util* — 계산 활용률. 100% 가까이 = 계산 병목
- *Memory-Usage* — VRAM 사용량
- *Power* — 현재/최대 전력
- *Temperature* — 80°C+ 면 thermal throttling

학습이 잘 되는데 *GPU-Util 30%* 면 *데이터 로딩 병목* 의심.

## 권 정리

- GPU = 처리량 최적화 (행렬 곱에 완벽)
- SM·Warp·Thread 계층

- Tensor Core = 행렬 곱 전용 하드웨어
- 자료형 = FP32/FP16/BF16/FP8/INT8
- 모델 메모리 계산이 핵심 역량
