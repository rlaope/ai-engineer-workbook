# Volume 95 — 데이터 거버넌스·라벨링

> 이 권이 끝나면 *AI 시스템의 가장 큰 약점은 데이터* 라는 명제를 시스템·정책·도구의 관점에서 다룰 수 있게 됩니다.

## 목적

데이터 거버넌스는 *데이터의 수집·저장·접근·보존·삭제* 의 정책과 시스템입니다. 라벨링은 *학습 데이터의 정답을 만드는 과정* 입니다. 둘 다 *AI 품질·규제 준수·비용* 모두에 결정적 영향을 줍니다.

## 선수 지식

- Volume 5, 27, 92 완료

## 학습 결과

1. 데이터 계약·접근 제어·감사 로그의 표준을 안다.
2. 라벨링 워크플로 (자동·반자동·인간·합성) 를 설계할 수 있습니다.
3. PII 처리·삭제 권리 (GDPR) 를 시스템에 반영할 수 있습니다.
4. 합성 데이터의 가능성과 한계를 안다.

---

## 1. 데이터 계약 (Data Contract)

(Vol 4 참고)

데이터 생산자와 소비자 간 *명시적 합의*: 스키마·SLA·변경 통보·책임자.

데이터 변경이 *조용히 모델을 망가뜨리는* 사고를 방지.

---

## 2. 접근 제어

### 2.1 RBAC (Role-Based Access Control)

```
역할: data-scientist, ml-engineer, analyst, admin
각 역할별 어느 데이터 어떤 작업 가능
```

### 2.2 컬럼 단위 권한

```sql
GRANT SELECT (id, name) ON users TO analyst;
-- email, phone 같은 PII 는 제외
```

### 2.3 데이터 마스킹

쿼리 시 자동으로 *PII 마스킹*. 민감한 영역은 *복호화 권한 별도*.

---

## 3. 감사 로그

*누가·언제·어떤 데이터를 봤는지* 모두 기록. 규제 (GDPR·HIPAA) 대응 필수.

---

## 4. PII 처리

### 4.1 식별자 분류

- *직접 식별자* — 이름·이메일·전화·주민번호
- *간접 식별자* — 생년월일+우편번호+성별 같은 조합

### 4.2 처리 옵션

- *가명화 (Pseudonymization)* — 식별자를 다른 ID 로
- *익명화 (Anonymization)* — 식별자 제거
- *차등 프라이버시 (DP)* — 통계적 노이즈
- *Federated Learning* — 데이터를 모으지 않음

---

## 5. 삭제 권리 (Right to be Forgotten)

GDPR·CCPA 의 핵심 권리. 사용자가 *자기 데이터 삭제 요청* 하면 시스템에서 제거.

도전:

- *학습된 모델에서 한 사용자만 제거* 는 매우 어려움 (Machine Unlearning 분야)
- *백업·로그* 까지 삭제 보장
- *모델 가중치에 잔여 정보* 위험

대응:
- 학습 시 *DP* 적용으로 개인 영향 최소화
- 정기 *재학습* 으로 옛 데이터 영향 감소
- *삭제 요청 추적 시스템*

---

## 6. 라벨링 워크플로

### 6.1 도구

- **Label Studio** — 오픈소스, 다양한 데이터 종류
- **Labelbox·Scale·SuperAnnotate** — SaaS

### 6.2 라벨러 종류

- *내부 직원* — 도메인 전문성, 비싸고 적은 양
- *크라우드소싱 (MTurk)* — 저렴하지만 품질 격차
- *전문 라벨링 회사* — 비싸지만 품질 보장
- *자동 라벨링 (LLM)* — 매우 저렴, 품질 검증 필수

### 6.3 품질 보장

- *다중 라벨러 + 합의* (3 명 이상이 같은 라벨)
- *황금 셋* — 알려진 정답으로 라벨러 평가
- *주기적 재교육*

---

## 7. 합성 데이터

### 7.1 발상

LLM 또는 시뮬레이터로 *학습 데이터 자동 생성*.

```python
# LLM 으로 (질문, 답변) 쌍 합성
for topic in topics:
    qa_pair = llm.complete(f"Generate a Q&A about {topic}")
    train_set.append(qa_pair)
```

### 7.2 강점

- *비용 매우 저렴*
- *프라이버시 부담 적음* (실제 데이터 아님)
- *희귀 케이스 의도적 생성*

### 7.3 한계

- *모델 붕괴 (Model Collapse)* — 합성 데이터로만 학습 시 분포 좁아짐
- *편향 증폭* — 생성 모델의 편향이 학습 데이터에 들어감

권장: *실제 데이터 + 합성 데이터 혼합*, 합성은 *부족한 부분 보완* 용.

---

## 8. 데이터 카드

(모델 카드와 비슷, 데이터에 대한 메타데이터)

```markdown
# Dataset Card: customer-feedback-2025

## Source
사내 고객센터 티켓, 2024 Q4

## Size
50,000 건

## Schema
- id, text, category, sentiment, timestamp

## PII
- 익명화 완료 (이름·이메일·전화 마스킹)

## Bias / Limitations
- 영어 80%, 한국어 15%, 기타 5%
- B2B 고객만 (B2C 제외)
```

## 권 정리

- 데이터 계약 = 생산자·소비자 합의
- 접근 제어 = RBAC + 컬럼 권한 + 마스킹
- 감사 로그 = 규제 대응
- PII 처리 = 가명화·익명화·DP·Federated
- 삭제 권리 = Machine Unlearning 도전
- 라벨링 = 도구·라벨러·품질 보장
- 합성 데이터 = 보완 용도, 모델 붕괴 주의
- 데이터 카드 = 표준 메타데이터
