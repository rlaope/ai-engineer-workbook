# Volume 102 — 데이터 거버넌스·라벨링

> 이 권이 끝나면 *AI 시스템의 가장 큰 약점은 데이터*라는 명제를 시스템·정책·도구의 관점에서 다룰 수 있게 됩니다.

## 목적

모델은 데이터의 거울입니다. 데이터에 편향이 있으면 모델에 편향이 생기고, 데이터에 PII 가 새면 모델 출력에서 PII 가 새어 나옵니다. 또한 학습 데이터는 *라벨링 워크플로* 의 산물이며, 라벨러의 일관성·QA·합성 데이터 활용이 모델 품질을 결정합니다. 이 권은 데이터 거버넌스와 라벨링의 표준 워크플로를 정리하며, 워크북의 마지막 권으로서 *AI 시스템의 평생 사이클*을 데이터 측면에서 마무리합니다.

## 선수 지식

- Volume 37, 50, 52 완료
- 외부 지식: 데이터 정책·개인정보 보호의 일반 개념

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. PII·민감 정보의 표준 정의와 처리 절차를 알 수 있습니다.
2. 라벨링 워크플로(가이드라인·라벨러·QA·계산자 합의) 를 설계할 수 있습니다.
3. 합성 데이터의 적법·윤리적 사용 기준을 갖습니다.
4. 데이터 카드(Datasheet for Datasets) 양식을 작성할 수 있습니다.
5. 데이터 보존·삭제·재학습 정책의 표준 양식을 갖습니다.

## 챕터 목차

1. **데이터 거버넌스의 정의** — 정책·소유·접근·감사
2. **PII 정의와 식별·익명화·재식별 위험**
3. **데이터 보존·삭제 정책**
4. **라벨링 워크플로** — 가이드라인·도구·QA
5. **계산자 합의 (Inter-Annotator Agreement)**
6. **합성 데이터 — 사용·평가·위험**
7. **Datasheet for Datasets** — 표준 데이터 카드
8. **GDPR·개인정보보호법의 큰 그림**
9. **재학습 트리거와 데이터 라이프사이클**
