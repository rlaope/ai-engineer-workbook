# Volume 54 — 모델 사이즈 의사결정

> 이 권이 끝나면 *어떤 작업에 어떤 크기의 모델을 써야 하는가* 의 의사결정 표가 머릿속에 그려지게 됩니다.

## 목적

가장 큰 모델이 항상 답은 아닙니다. 같은 작업을 7B 모델로 해결할 수 있다면 70B 모델은 비용·지연만 키울 뿐입니다. 모델 캐스케이딩(작은 모델 → 큰 모델로 라우팅), Mixture of Experts, 작업별 전문화 모델 같은 전략이 *비용 대비 성능*의 황금 영역을 만듭니다. 이 권은 모델 사이즈 의사결정의 사고 도구를 정리합니다.

## 선수 지식

- Volume 63, 48 완료
- 외부 지식: 비용/성능 트레이드오프의 직관

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. 작업 난이도를 모델 사이즈에 매핑하는 표를 작성할 수 있습니다.
2. 모델 캐스케이딩 패턴을 그릴 수 있습니다.
3. 라우터 모델의 학습·운영 방법을 알 수 있습니다.
4. 작은 모델로 큰 모델을 *대체할 수 있는 작업*을 식별할 수 있습니다.
5. 사이즈 결정의 *환경 영향* 을 인식합니다.

## 챕터 목차

1. **모델 사이즈와 능력의 관계**
2. **작업 난이도 분류** — 추출·요약·분류·추론·창작
3. **모델 캐스케이딩** — Cheap → Expensive
4. **라우터 모델** — 작업·복잡도 기반 분기
5. **소형 모델의 강점** — 미세조정·온디바이스
6. **MoE 의 비용 모델 (Vol 53 와 연결)**
7. **상용 vs 오픈소스 의사결정 매트릭스**
8. **환경 영향과 책임**
