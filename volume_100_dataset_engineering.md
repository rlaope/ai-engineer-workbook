# Volume 91 — 데이터셋 엔지니어링 종합

> 이 권이 끝나면 *데이터의 양보다 데이터의 큐레이션이 모델 품질을 결정한다* 는 명제를 시스템 단위로 다룰 수 있게 됩니다.

## 목적

학습 데이터의 *품질·다양성·중복 제거·합성 비율* 이 모델의 성능을 좌우합니다. LIMA·Alpaca·Phi 시리즈는 *적은 양의 고품질 데이터*가 큰 양의 평범한 데이터를 능가한다는 것을 증명했습니다. 이 권은 데이터 큐레이션·합성·증류·검사·중복 제거의 표준 워크플로를 정리합니다.

## 선수 지식

- Volume 37, 86 완료
- 외부 지식: ETL·데이터 파이프라인의 일반 개념

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. 데이터 품질의 5 가지 축(정확성·완전성·일관성·시의성·관련성) 을 측정할 수 있습니다.
2. 데이터 합성 기법(템플릿·LLM 합성·증류) 을 적용할 수 있습니다.
3. MinHash·SimHash 로 중복 데이터를 찾을 수 있습니다.
4. n-gram·임베딩 기반 데이터 정리 파이프라인을 설계할 수 있습니다.
5. 데이터 카드(Datasheet) 를 작성할 수 있습니다.

## 챕터 목차

1. **데이터 큐레이션의 영향력**
2. **데이터 품질 측정 5 축**
3. **데이터 양 vs 품질 — Phi·LIMA 의 교훈**
4. **데이터 합성 — 템플릿·자가 지시(Self-Instruct)·LLM 합성**
5. **모델 증류 — Teacher → Student 데이터 생성**
6. **중복 제거 — 정확·근사(MinHash·SimHash)**
7. **데이터 정리 — 길이·언어·toxicity·PII 필터**
8. **데이터 형식 정규화** — JSONL·Chat 템플릿
9. **데이터 카드와 메타데이터**

## 자가점검 키워드
