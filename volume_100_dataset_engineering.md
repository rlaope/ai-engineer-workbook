## 1. 데이터 큐레이션

### 1.1 절차

```
1. 수집 — 어디서 어떻게 모을지
2. 검사 — 품질·다양성·편향
3. 필터링 — 저품질·중복·유해 제거
4. 정리 — 형식·라벨 표준화
5. 분할 — 학습/검증/테스트
6. 버저닝 — DVC 등으로 추적
```

각 단계가 *모델 품질에 직접 영향*.

### 1.2 품질 지표

- *라벨 정확도*
- *분포 균형* (클래스·도메인·시간)
- *중복도*
- *결측·오류 비율*
- *대표성* (실제 사용자 분포와 일치)

## 3. 중복 제거

### 3.1 중요성

LLM 사전학습에서 *같은 문서가 반복* 되면 *암기·과적합*. *중복 제거가 모델 품질에 큰 영향*.

### 3.2 기법

- **정확 중복** — Hash 기반
- **유사 중복** — MinHash, SimHash
- **시멘틱 중복** — 임베딩 유사도

```python
from datasketch import MinHash

def get_minhash(text):
    m = MinHash()
    for word in text.split():
        m.update(word.encode())
    return m

# 두 문서의 유사도
similarity = m1.jaccard(m2)
```

## 5. 모델 증류 (Knowledge Distillation)

큰 모델 (Teacher) 의 출력을 *작은 모델 (Student) 이 모방*. 합성 데이터 + 증류가 *작은 모델 미세조정* 의 표준.

```
GPT-4 (Teacher) → 100K 응답 생성
                ↓
LLaMA 3 8B (Student) → 그 데이터로 미세조정
                     ↓
         GPT-4 의 80% 정확도 + 1/100 비용
```

## 6. 데이터 정리·필터링

### 6.1 자동 필터

- *너무 짧은 텍스트* (< 10 단어)

## 7. Data-Centric 사고

### 7.1 발상

같은 모델 코드·같은 시간 예산에서, *데이터 개선이 모델 개선보다 큰 효과*.

### 7.2 실천

- *모델 변경 횟수 < 데이터 변경 횟수*
- *데이터 검토 회의* 정기 운영
- *라벨링 일관성 추적*
