# Volume 63 — LLM 사전학습과 스케일링 법칙

> 이 권이 끝나면 *LLM 이 어떻게 학습되고 왜 더 키울수록 좋아지는가* 를 데이터·계산·파라미터의 세 변수로 설명할 수 있게 됩니다.

## 목적

LLM 의 사전학습은 *방대한 텍스트로 다음 토큰을 맞추는 자가지도 학습* 이라는 한 문장으로 요약됩니다. 이 단순한 목표가 모델 크기·데이터·계산을 늘릴수록 *어떤 비례 법칙* 에 따라 일관되게 좋아진다는 발견 (스케일링 법칙) 이 GPT 시리즈와 LLaMA 계보를 만든 동력입니다.

## 1. 다음 토큰 예측

### 1.1 학습 목표

LLM 사전학습 = *주어진 토큰들로 다음 토큰의 확률 분포 예측*.

```
입력: "The quick brown fox jumps"
정답: "over"

손실: -log P("over" | "The quick brown fox jumps")
```

라벨링 비용 0 — *데이터 자체에서 정답 추출* (자가지도 학습).

### 1.2 단순함의 힘

이 단순한 목표가 *번역·요약·코드·추론·창작* 같은 모든 능력을 동시에 학습시킵니다. *충분히 많은 텍스트를 읽으면 그 안에 모든 작업이 자연스럽게 들어 있기* 때문.

---

## 2. 스케일링 법칙

### 2.1 카플란 법칙 (2020)

OpenAI 의 발견: *모델 크기·데이터·계산* 을 멱법칙으로 늘리면 *손실이 멱법칙으로 감소*.

함의: *더 큰 모델 + 더 많은 계산 = 더 좋은 모델*.

### 2.2 친칠라 법칙 (2022)

DeepMind 의 후속 연구. *주어진 컴퓨팅 예산에서 최적 모델 크기와 데이터 양의 비율* 을 구체화.

핵심 결론: *모델 파라미터 1 개당 약 20 토큰* 이 최적.

GPT-3 (175B) 는 *과대 매개화* — 데이터가 부족했음. 친칠라 법칙으로 학습된 70B 모델이 GPT-3 보다 좋았음.

이 발견 후 LLaMA 같은 모델이 *더 작지만 더 많은 데이터* 로 학습.

## 3. 모델 계보

### 3.1 GPT 시리즈 (OpenAI)

- GPT-1 (2018) — 117M
- GPT-2 (2019) — 1.5B
- GPT-3 (2020) — 175B
- GPT-3.5 (2022) — InstructGPT
- GPT-4 (2023) — 멀티모달, MoE 추정
- GPT-4o, o1, o3 (2024-2025)

### 3.2 LLaMA 시리즈 (Meta)

- LLaMA 1 (2023) — 7B/13B/33B/65B
- LLaMA 2 (2023) — 7B/13B/70B, 상업 사용 허용
- LLaMA 3 (2024) — 8B/70B/405B
- LLaMA 4 (2025+)

오픈 모델의 표준.

### 3.3 Mistral

- Mistral 7B (2023) — 작지만 강함
- Mixtral 8x7B (2023) — MoE
- Codestral (2024) — 코드 특화

### 3.4 DeepSeek·Qwen 등

DeepSeek-V3 (2024), Qwen 2.5 (2024) 등 *오픈 모델의 정상* 위치를 차지.

## 4. 학습 데이터

### 4.1 출처

- 웹 크롤링 (Common Crawl)
- 도서·논문
- 코드 (GitHub)
- 위키
- 합성 데이터 (LLM 으로 생성)

### 4.2 품질·중복·다양성

- *중복 제거* — 같은 문서가 여러 번 학습되면 *암기·과적합*
- *품질 필터링* — 스팸·자동 생성·저품질 텍스트 제거
- *다양성* — 도메인·언어·작성 스타일의 균형

데이터 품질이 *모델 품질의 상한선* 을 결정.

---

## 5. 학습 인프라

대형 LLM 학습은:
- 수천-수만 GPU (H100 클러스터)
- 수개월 학습 시간
- 분산 학습 (FSDP·텐서/파이프라인 병렬)
- 수백만-수억 달러 비용

이 영역은 ML 엔지니어·인프라 엔지니어의 영역이며, AI 엔지니어는 *결과 모델을 가져다 사용*.
