# Volume 20 — 머신러닝의 본질과 학습 패러다임

> 이 권이 끝나면 새로운 ML 문제를 만났을 때 *어떤 패러다임에 속하는가* 를 즉시 분류할 수 있게 됩니다.

## 목적

머신러닝은 단일 알고리즘의 모음이 아니라 *데이터로부터 패턴을 학습하는 프레임워크* 입니다. 어떤 데이터·신호가 주어지는가에 따라 학습 패러다임이 달라지고, 패러다임이 정해지면 사용 가능한 알고리즘과 평가 방식이 자연스럽게 결정됩니다. 이 권은 학습 패러다임의 큰 분류부터 잡고, 이후 권에서 다룰 구체적 알고리즘들이 어디에 위치하는지 미리 표시합니다.

## 1. 머신러닝이란 무엇인가

### 1.1 정의

머신러닝은 *명시적으로 프로그래밍되지 않고도 데이터로부터 학습할 수 있는 능력을 컴퓨터에 부여하는 연구 분야* 입니다 (Arthur Samuel, 1959).

전통적 프로그램과의 차이:

```
[전통적 프로그램]
입력 + 규칙 (사람이 작성) → 출력

[머신러닝]
입력 + 출력 (데이터) → 규칙 (모델이 학습)
```

이 단순한 시각의 전환이 *복잡한 규칙을 사람이 직접 작성하기 어려운 문제* 의 해결을 가능하게 만들었습니다.

### 1.2 챕터 정리

머신러닝은 *데이터로부터 규칙을 학습* 하는 프레임워크이며, 전통적 프로그래밍의 *규칙을 사람이 작성* 하는 방식과 본질적으로 다릅니다.

---

## 2. 학습 패러다임 4 가지

### 2.1 지도 학습 (Supervised Learning)

*입력과 정답 라벨* 쌍이 주어집니다. 모델은 *입력 → 라벨* 의 매핑을 학습합니다.

- 회귀 — 라벨이 *연속 값* (예: 집 가격)
- 분류 — 라벨이 *이산 클래스* (예: 스팸/정상)

데이터 양: *라벨링 비용이 비싸* 일반적으로 작음 (수만-수백만 건).

### 2.2 비지도 학습 (Unsupervised Learning)

*입력만* 주어지고 라벨은 없습니다. 모델은 *데이터의 구조* 를 학습합니다.

- 군집화 (Clustering)
- 차원 축소
- 이상 탐지
- 밀도 추정

### 2.3 자가지도 학습 (Self-Supervised Learning)

*라벨이 없지만, 데이터 자체에서 학습 신호를 만듭니다*.

대표 사례: LLM 의 *다음 토큰 예측* — 텍스트 자체에서 *앞 토큰들 → 다음 토큰* 의 매핑을 학습.

데이터 양: *라벨이 필요 없어* 매우 큼 (수십억 토큰).

이 패러다임이 *파운데이션 모델 시대* 를 가능하게 한 핵심입니다.

### 2.4 강화 학습 (Reinforcement Learning)

*환경과 상호작용* 하면서 *보상 신호* 로 학습합니다.

- 행동 → 환경 응답 → 보상
- *누적 보상 최대화* 를 목표로 정책 학습

대표 사례: AlphaGo, AlphaZero, RLHF (LLM 정렬).

### 2.5 챕터 정리

4 가지 패러다임은 *어떤 신호로부터 학습하는가* 의 차이로 분류됩니다. 새 문제를 만나면 *데이터에 무엇이 있는가* 를 먼저 봅니다.

---

## 3. 학습/검증/테스트 분할

### 3.1 세 분할의 목적

- *학습 셋* — 모델 가중치 학습용
- *검증 셋* — 하이퍼파라미터 선택·조기 종료
- *테스트 셋* — 최종 일반화 성능 측정

비율 권장: 80/10/10 또는 70/15/15.

### 3.2 데이터 누수 방지

테스트 셋이 *학습 또는 검증 셋과 겹치면* 평가가 의미 없습니다. *시간 순서·해시 기반* 분할로 방지.

### 3.3 챕터 정리

세 분할은 *역할이 모두 다름* 을 명심해야 합니다. 테스트 셋은 *최종 평가 한 번만* 사용합니다.

---

## 4. 편향-분산 트레이드오프

### 4.1 편향과 분산

- *편향 (Bias)* — 모델이 *너무 단순해서* 데이터의 패턴을 못 잡음 (과소적합)
- *분산 (Variance)* — 모델이 *너무 복잡해서* 학습 데이터에 과적합 (과대적합)

총 오차 = 편향² + 분산 + 환원 불가능 노이즈

### 4.2 모델 복잡도와의 관계

```
오차
 |
 |\           __ 총 오차
 | \         /
 |  \       /
 |   \  __ /  __ 분산
 |    \/  \/
 |    /\  /\
 |   /  \/  \
 |  /   /\   \__ 편향²
 | /   /  \
 |____|________
   ↑    모델 복잡도
   최적 지점
```

### 4.3 챕터 정리

편향-분산 트레이드오프는 *모델 선택의 핵심 트레이드오프* 이며, 검증 셋 성능으로 *최적 복잡도* 를 찾습니다.

---

## 5. No Free Lunch 정리

*모든 가능한 데이터 분포* 에 대해 *평균적으로* 모든 알고리즘은 *동등한 성능* 을 보입니다 (Wolpert, 1996).

함의: *특정 도메인에서 잘 작동하는 알고리즘은 다른 도메인에서는 못할 수 있다*. *모든 문제에 좋은 알고리즘* 은 없습니다.

실무적 의미: *알고리즘 선택은 도메인 특성에 의존* 합니다. 자기 도메인에 맞는 알고리즘을 *경험적으로* 찾아야 합니다.

## 권 정리

- 머신러닝 = 데이터로부터 규칙 학습
- 4 패러다임: 지도·비지도·자가지도·강화
- 세 분할: 학습·검증·테스트
- 편향-분산 트레이드오프 = 모델 복잡도 선택의 핵심
- No Free Lunch — 만능 알고리즘 없음

가장 기억에 남겨야 할 한 줄은 **"새 ML 문제는 패러다임 분류 → 알고리즘 선택 → 분할 → 평가의 순서로 접근한다."** 입니다.
