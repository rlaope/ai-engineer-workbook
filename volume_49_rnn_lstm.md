## 1. RNN

### 1.1 구조

```
x_t → [RNN 셀] → h_t
           ↑
         h_{t-1}
```

각 시점 출력 $h_t = \tanh(W_h h_{t-1} + W_x x_t + b)$.

이전 상태가 *순환 (Recurrent)* 으로 다음 단계에 전달.

### 1.2 PyTorch

```python
import torch.nn as nn
rnn = nn.RNN(input_size=10, hidden_size=32, num_layers=2)
```

---

## 2. 그래디언트 소실

긴 시퀀스에서 *역전파 시 같은 가중치 행렬이 반복 곱해져* 그래디언트가 0 으로 수렴 (또는 폭발).

함의: *5-10 스텝 이상 떨어진 정보를 학습 못함*.

---

## 3. LSTM (1997)

### 3.1 게이트 메커니즘

*세 게이트* 로 정보의 흐름 제어:

- **Forget gate** — 이전 상태에서 무엇을 잊을지
- **Input gate** — 새 정보 중 무엇을 저장할지
- **Output gate** — 무엇을 출력할지

```python
lstm = nn.LSTM(input_size=10, hidden_size=32, num_layers=2)
```

게이트 구조 덕분에 *수십-수백 스텝의 의존성* 학습 가능.

### 3.2 GRU

LSTM 의 단순화. *2 게이트* 로 비슷한 성능.

---

## 4. RNN 의 본질적 한계

### 4.1 순차 계산

각 시점이 *이전 시점의 결과* 에 의존. *병렬화 불가능*. GPU 의 강점을 활용 못함.

### 4.2 긴 시퀀스의 어려움

LSTM 도 *수천 토큰* 길이의 의존성은 어려움.

### 4.3 정보 병목

이전 모든 정보가 *고정 크기 hidden state* 에 압축. 손실 발생.

---

## 5. Attention 의 등장 → Transformer

2014 년 *Attention 메커니즘* 등장. RNN 에 *직접 과거 토큰 참조* 능력 추가.

2017 년 Transformer — *RNN 자체를 버리고 Attention 만으로 시퀀스 처리*. 병렬 가능. 긴 의존성 잘 잡음.

이후 거의 모든 시퀀스 모델이 트랜스포머로 이동.

---

## 6. 여전히 RNN 이 적합한 경우

- *매우 긴 시퀀스* (수백만 토큰) — 트랜스포머의 O(n²) 가 불가능
- *온디바이스 추론* (작은 메모리)
- *시계열 예측 (특정 도메인)*
- *State Space Model (Mamba 등)* — RNN 의 부활 형태

---

## 권 정리

- RNN = 순환 구조, 시퀀스 모델의 첫 표준
- LSTM = 게이트로 그래디언트 소실 완화
- 한계: 순차 계산·긴 의존성·정보 병목
- 트랜스포머로 대체됨 (2017+)
- Mamba 등 State Space Model 로 RNN 부활

가장 기억할 한 줄: **"RNN 은 순차 계산의 한계로 트랜스포머에 자리를 내줬지만, 매우 긴 시퀀스에서는 SSM 으로 부활했다."**
