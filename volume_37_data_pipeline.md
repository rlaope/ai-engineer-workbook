# Volume 63 — 데이터 파이프라인과 실험 관리

> 이 권이 끝나면 *재현 가능한 실험* 을 설계할 수 있게 됩니다.

## 목적

학습이 한 번 잘된 모델은 의미가 없습니다. 같은 코드를 다른 사람이 돌렸을 때, 또는 한 달 뒤의 자신이 다시 돌렸을 때 같은 결과가 나오는 *재현성*이 ML 엔지니어링의 핵심 덕목입니다. 데이터 파이프라인의 결정성·시드 고정·실험 추적·아티팩트 버저닝이 그 재현성을 만듭니다. 이 권은 실험 관리 도구의 사고법을 다집니다.

## 선수 지식

- Volume 36 완료
- 외부 지식: git 사용 경험

## 학습 결과

이 권을 마치면 다음을 할 수 있게 됩니다.

1. 데이터 파이프라인의 결정성을 보장하는 시드 정책을 작성할 수 있습니다.
2. `Dataset` 을 효율적으로 만들기 위한 캐싱·shard·prefetch 전략을 알 수 있습니다.
3. W&B 또는 MLflow 로 실험을 추적하고 비교할 수 있습니다.
4. 모델·데이터·코드의 *세 가지 버전*을 함께 기록하는 실험 노트 양식을 설계할 수 있습니다.
5. 실험 폭발(같은 실험을 100 번 돌리는 상황) 을 통제할 수 있습니다.

## 챕터 목차

1. **재현성의 정의** — 무엇이 같으면 같은 실험인가
2. **시드 고정 정책** — Python·NumPy·PyTorch·CUDA
3. **`Dataset` 효율화** — 메모리 매핑·shard·prefetch
4. **데이터 검증** — 학습 시작 전에 깨진 샘플 잡기
5. **W&B / MLflow 사용 패턴** — 실험·런·아티팩트
6. **하이퍼파라미터 탐색** — 그리드·랜덤·베이지안·HyperBand
7. **모델 카드와 데이터셋 카드** — 메타데이터의 표준
8. **실험 노트 양식 설계**
