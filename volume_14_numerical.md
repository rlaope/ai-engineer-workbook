## 1. IEEE 754 의 구조

### 1.1 부동소수점의 세 부분

부동소수점 숫자는 *부호 비트 + 지수 + 가수* 의 세 부분으로 표현됩니다.

```
FP32 (32 비트):
[부호 1][지수 8][가수 23]

값 = (-1)^부호 × 1.가수 × 2^(지수-127)
```

지수는 *바이어스* (FP32 의 경우 127) 로 표현되어 *음의 지수* 도 표현 가능합니다. 가수는 *암묵적 1* 을 포함합니다 (정규화).

### 1.2 자료형 비교

```
+--------+------+------+------+------------+------------+
| 자료형 | 부호 | 지수 | 가수 | 표현 범위  | 정밀도     |
+--------+------+------+------+------------+------------+
| FP32   |  1   |  8   |  23  | ±10^38     | 7 자리수  |
| FP16   |  1   |  5   |  10  | ±65,504    | 3 자리수  |
| BF16   |  1   |  8   |  7   | ±10^38     | 2-3 자리수|
| FP8 E4M3|  1  |  4   |  3   | ±240       | 1 자리수  |
+--------+------+------+------+------------+------------+
```

**FP16** 는 *지수 비트가 적어 표현 범위가 좁습니다* (±65,504). 큰 그래디언트가 *오버플로* 로 무한대가 되기 쉬움.

**BF16** 는 *지수가 FP32 와 동일* 해 표현 범위는 같지만 *정밀도가 낮습니다*. 학습에서 표현 범위가 정밀도보다 중요한 경우가 많아 *현대 LLM 학습의 표준* 이 됨.

**FP8** 는 H100 부터 지원되는 *추론·학습 가속용*. 정밀도가 매우 낮아 *세심한 스케일링* 이 필수.

### 1.3 챕터 정리

IEEE 754 는 부호·지수·가수 세 부분이며, 자료형별 *표현 범위와 정밀도* 의 트레이드오프가 학습 안정성을 결정합니다.

## 2. 반올림 오차의 누적

### 2.1 같은 식이 다른 결과

```python
import numpy as np

# 더하기 순서가 결과를 바꿈
a = np.float32(1e10)
b = np.float32(1.0)
c = np.float32(-1e10)

print((a + b) + c)   # 0.0  (1.0 이 손실)
print(a + (b + c))   # 1.0  (정상)
```

큰 수와 작은 수를 더할 때 *작은 수가 흡수* 되는 *catastrophic cancellation* 입니다.

### 2.2 학습에서의 영향

100 만 개 샘플의 손실을 누적할 때, *순서에 따라* 미묘하게 다른 결과가 나옵니다. *Kahan summation* 같은 보상 알고리즘으로 정확도를 올릴 수 있지만, 일반적으로 *충분히 큰 평균* 은 안정적이라 무시됩니다.

### 2.3 챕터 정리

부동소수점 덧셈은 *결합 법칙* 이 성립하지 않으며, 큰 수+작은 수 조합에서 정보 손실이 발생합니다.

## 3. 조건수 — 수치적 민감도

### 3.1 조건수의 의미

행렬 A 의 *조건수* $\kappa(A) = \|A\| \cdot \|A^{-1}\|$ 는 *입력 변화가 출력에 얼마나 증폭되는가* 를 측정합니다.

- 조건수 1 — 가장 안정적
- 조건수 10^k — k 자리수의 정밀도 손실

```python
import numpy as np

A = np.array([[1, 1], [1, 1.001]])
print(np.linalg.cond(A))   # ~4000

# 작은 입력 변화가 큰 출력 변화로
b1 = np.array([2, 2])
b2 = np.array([2, 2.001])
print(np.linalg.solve(A, b1))   # [2, 0]
print(np.linalg.solve(A, b2))   # 큰 차이
```

### 3.2 신경망 가중치의 조건수

학습이 진행되면서 가중치 행렬의 조건수가 *폭증* 하면 학습이 불안정해집니다. *가중치 정칙화·정규화 (BatchNorm 등)* 가 조건수를 통제하는 도구입니다.

### 3.3 챕터 정리

조건수는 *수치 민감도* 의 측정이며, 큰 조건수는 *학습 불안정* 의 신호입니다.

## 4. 수치 안정한 알고리즘

### 4.1 LogSumExp

소프트맥스 계산:

$$\text{softmax}(z)_k = \frac{\exp(z_k)}{\sum_j \exp(z_j)}$$

큰 z 값이 있으면 `exp(z)` 가 오버플로. 안정화:

$$\text{softmax}(z)_k = \frac{\exp(z_k - \max z)}{\sum_j \exp(z_j - \max z)}$$

```python
def stable_softmax(z):
    z = z - z.max()
    e = np.exp(z)
    return e / e.sum()
```

### 4.2 Kahan 합산

큰 수의 누적 합 정확도 향상:

```python
def kahan_sum(arr):
    total = 0.0
    c = 0.0  # 보상
    for x in arr:
        y = x - c
        t = total + y
        c = (t - total) - y
        total = t
    return total
```

NumPy 의 `np.sum` 은 자동으로 *pairwise summation* 을 사용해 단순 누적보다 안정적이지만, 매우 정밀한 계산이 필요하면 Kahan 사용.

### 4.3 챕터 정리

LogSumExp·Kahan 합산 같은 *수치 안정한 알고리즘* 이 학습의 안정성을 만듭니다.

## 5. NaN/Inf 디버깅 워크플로

### 5.1 발생 원인 5 가지

1. *학습률 과다* → 그래디언트 폭발
2. *log(0) 또는 1/0* → 무한대
3. *입력에 NaN/Inf* → 전파
4. *softmax overflow* → exp 오버플로
5. *FP16 의 underflow* → 작은 그래디언트가 0

### 5.2 진단 절차

```python
# 입력 검증
assert torch.isfinite(batch).all(), "Input has NaN/Inf"

# 매 스텝 손실 검증
loss = compute_loss(...)
if not torch.isfinite(loss):
    # 마지막 정상 체크포인트로 롤백
    ...

# 그래디언트 노름 모니터링
total = sum(p.grad.norm() for p in model.parameters() if p.grad is not None)
print(f"Grad norm: {total:.4f}")
```

### 5.3 챕터 정리

NaN/Inf 디버깅은 *입력 검증·손실 검증·그래디언트 모니터링* 의 3 단계로 진행합니다.
