## 1. 분류 메트릭

### 1.1 혼동 행렬 기반

```
              예측 양성  예측 음성
실제 양성     TP         FN
실제 음성     FP         TN
```

- *정확도* = (TP+TN)/전체
- *정밀도* = TP/(TP+FP) — *FP 를 낮추는 것 중요*
- *재현율* = TP/(TP+FN) — *FN 을 낮추는 것 중요*
- *F1* = 정밀도와 재현율의 조화평균

### 1.2 ROC-AUC vs PR-AUC

- **ROC-AUC** — 모든 임계값에서의 (TPR, FPR) 곡선 아래 면적. 균형 데이터에 적합.
- **PR-AUC** — (Precision, Recall) 곡선 아래 면적. *클래스 불균형* 에 더 적합.

```python
from sklearn.metrics import roc_auc_score, average_precision_score

roc = roc_auc_score(y_true, y_score)
pr = average_precision_score(y_true, y_score)
```

### 1.3 클래스 불균형의 함정

99% 음성·1% 양성에서 *모두 음성으로 예측* 하면 정확도 99%. 의미 없음. *PR-AUC·F1·Recall* 사용.

## 2. 회귀 메트릭

```
+-------+------------------+----------------+
| 지표  | 정의             | 강점           |
+-------+------------------+----------------+
| MAE   | mean(|y - ŷ|)    | 이상치 강건    |
| MSE   | mean((y - ŷ)²)   | 큰 오차에 민감 |
| RMSE  | √MSE              | 단위 일치      |
| MAPE  | mean(|y - ŷ|/|y|) | 비율 해석      |
| R²    | 분산 설명력      | 0-1 범위       |
+-------+------------------+----------------+
```

선택: *이상치 많음 → MAE*, *큰 오차 페널티 → MSE/RMSE*, *해석 쉬움 → R² 또는 RMSE*.

---

## 3. 랭킹 메트릭

추천·검색에서 사용:

- **MRR (Mean Reciprocal Rank)** — 첫 정답 위치의 역수 평균
- **NDCG@k** — 상위 k 의 품질 (위치 가중)
- **MAP** — Mean Average Precision
- **Hit@k** — 상위 k 안에 정답 있는가

---

## 4. 생성 모델 메트릭

### 4.1 텍스트

- **BLEU** — n-gram 겹침 (번역)
- **ROUGE** — 요약 평가
- **BERTScore** — 임베딩 유사도
- **METEOR** — BLEU + 동의어 처리
- **Perplexity** — 언어 모델 자체 평가

### 4.2 이미지

- **FID (Fréchet Inception Distance)** — 생성 vs 실제 분포 거리
- **Inception Score** — 다양성·품질
- **CLIP Score** — 텍스트-이미지 일치

### 4.3 LLM 평가의 어려움

생성 텍스트는 *정답이 여러 개* 입니다. BLEU·ROUGE 는 *표면적 유사도* 만 측정. 진짜 품질을 측정하려면:

- **인간 평가** — 비용 큼
- **LLM-as-Judge** — 다른 LLM 으로 평가. GPT-4 가 평가자로 표준
- **Arena (Pairwise Preference)** — 두 모델 응답 비교

---

## 5. 비용 가중 메트릭

도메인별로 *FP 와 FN 의 비용이 다름*:

- 의료 진단: FN (놓친 병) >> FP (오진)
- 스팸 필터: FP (정상을 스팸으로) > FN (스팸 통과)
- 신용 평가: FP (좋은 사람 거절) ≈ FN (나쁜 사람 승인)

비용 가중 메트릭:

$$\text{Cost} = c_{FP} \cdot N_{FP} + c_{FN} \cdot N_{FN}$$

이 비용을 *모델 학습 손실에 직접 반영* 하거나 *임계값 조정* 으로 최적화.
