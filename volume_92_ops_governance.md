# Volume 92 — AI 시스템의 운영·관측·비용·거버넌스

> 이 권이 끝나면 *AI 시스템을 운영한다* 는 말이 *모델을 띄운다* 가 아니라 *모델·데이터·정책의 평생 사이클을 지탱한다* 임을 알게 됩니다.

## 목적

학습된 모델을 띄우는 일은 시작에 불과합니다. 데이터 분포는 시간이 지나면 변하고, 모델은 점차 부정확해지며, 비용은 누적됩니다. 또한 안전·공정성·프라이버시·규제 같은 *기술 외 요구사항* 이 함께 따라옵니다. 이 권은 AI 시스템의 운영 사이클 전체를 다룹니다.

## 선수 지식

- Volume 5, 38, 86 완료

## 학습 결과

1. 모델 모니터링의 4가지 축 (성능·드리프트·데이터 품질·비용) 을 설명할 수 있습니다.
2. A/B 테스트·다단계 롤아웃·섀도우 배포의 차이를 알 수 있습니다.
3. AI 시스템의 비용 모델 (GPU·스토리지·전송·인간 평가) 을 분해할 수 있습니다.
4. 안전·공정성·프라이버시·규제 의 표준 점검 항목을 인식합니다.
5. 인시던트 사후 분석을 모델·데이터·정책 모두에 대해 수행할 수 있습니다.

---

## 1. 모델의 수명 주기

```
[학습] → [평가] → [배포] → [모니터링] → [드리프트 감지] → [재학습] → ...
```

배포된 모델은 *영원히 살지 않습니다*. 데이터·환경·요구사항이 변하면서 *조용히 정확도가 떨어집니다*. 운영의 본질은 이 *조용한 감쇠* 를 감지·대응하는 일.

## 2. 모델 모니터링 4축

### 2.1 성능

- 정확도·F1·ROC-AUC 등 메트릭
- *라벨이 즉시 도착하는 경우* 만 직접 측정 가능

### 2.2 드리프트

- *데이터 드리프트* — 입력 분포 변화
- *컨셉 드리프트* — 입력→출력 관계 변화
- *예측 드리프트* — 출력 분포 변화

PSI·KS·MMD 같은 통계로 측정.

### 2.3 데이터 품질

- 결측·이상치
- 스키마 변경
- 신선도 (지연)

### 2.4 비용

- GPU 시간
- API 토큰
- 인간 평가
- 데이터 저장·전송

## 3. A/B 테스트·롤아웃·섀도우

### 3.1 A/B 테스트

사용자를 *두 그룹* 으로 나눠 *다른 모델 버전* 노출. 비즈니스 메트릭 차이 측정.

### 3.2 다단계 롤아웃

5% → 25% → 50% → 100% 점진. 각 단계에서 메트릭 점검.

### 3.3 섀도우 배포

새 모델을 *실제 트래픽으로 처리하지만 응답은 옛 모델 사용*. 성능·비용 사전 측정.

각 패턴이 *다른 위험 통제* 단계.

## 10. 인시던트 사후 분석

표준 양식:

```
1. 무엇이 일어났는가
2. 영향 범위
3. 발견 시점·방법
4. 대응 흐름
5. 근본 원인
6. 재발 방지
```

비난 없는 (blameless) 문화로 *팀의 학습 자산* 으로 활용.
