# Volume 97 — 사용자 피드백 시스템

> 이 권이 끝나면 *모델을 시간이 갈수록 더 좋아지게 만드는 피드백 루프* 를 시스템 단위로 설계할 수 있게 됩니다.

## 목적

배포된 모델은 사용자와 만나며 *명시적 피드백 (별점·👍/👎)·암시적 피드백 (클릭·이탈·재시도)* 을 끊임없이 만들어 냅니다. 이 신호를 다시 학습 데이터로 환원하는 루프가 *데이터 플라이휠* 이며, 좋은 AI 제품과 평범한 AI 제품을 가르는 가장 큰 차이입니다.

## 선수 지식

- Volume 92, 93 완료

## 학습 결과

1. 명시·암시 피드백의 차이와 수집 비용을 비교할 수 있습니다.
2. 대화형 피드백 추출 (Turn-level·Session-level) 양식을 설계할 수 있습니다.
3. 편향 (자기 선택·생존자) 을 인식하고 보정할 수 있습니다.
4. 피드백 → SFT/DPO 데이터로의 변환 파이프라인을 설계할 수 있습니다.
5. *데이터 플라이휠* 을 자기 제품에 적용할 수 있습니다.

---

## 1. 피드백의 두 종류

### 1.1 명시적 피드백

- 👍/👎 버튼
- 별점 (1-5)
- 짧은 자유 텍스트
- 신고 버튼

### 1.2 암시적 피드백

- 응답 복사
- 응답 삭제·재생성
- 대화 종료 후 재방문
- 체류 시간

```
+----------+----------+----------+
| 종류     | 정확도   | 수집비용 |
+----------+----------+----------+
| 명시     | 높음     | 높음 (마찰)|
| 암시     | 중간     | 낮음 (자동)|
+----------+----------+----------+
```

산업에서는 *둘 다 사용*. 명시는 적은 양으로 정확, 암시는 대량으로 균형.

---

## 2. 수집 UI 설계

### 2.1 마찰 최소화

- 한 클릭 (👍/👎)
- 모달 없이 인라인
- 선택적 텍스트

### 2.2 Turn-level vs Session-level

- *Turn-level* — 각 응답마다 피드백
- *Session-level* — 대화 전체에 한 번

Turn-level 이 *세밀한 데이터*, Session-level 이 *마찰 적음*. 둘을 병행.

---

## 3. 편향과 노이즈

### 3.1 자기 선택 편향

피드백을 *주는 사용자가 평균과 다름*. 매우 만족하거나 매우 불만족한 사용자만 응답.

### 3.2 생존자 편향

이탈한 사용자는 *피드백을 안 주고 떠남*. 데이터에 *남은 사용자만* 반영.

### 3.3 보정

- *이탈 사용자도 데이터로* (마지막 응답 = 부정 신호 가설)
- *피드백 응답률 모니터링*
- *대표성 있는 표본* 으로 가중

---

## 4. 피드백 → 학습 데이터

### 4.1 SFT 데이터

👍 응답을 *Supervised Fine-Tuning* 데이터로:

```
(prompt, 👍 응답)
```

### 4.2 DPO 데이터

👍 와 👎 비교를 *DPO 학습 데이터* 로:

```
(prompt, chosen=👍, rejected=👎)
```

### 4.3 Reward Model 학습

대규모 비교 데이터로 *Reward Model* 학습 → RLHF.

---

## 5. 데이터 플라이휠

```
사용자 → 모델 응답 → 피드백 → 학습 데이터 → 모델 개선 → 더 좋은 응답 → 더 많은 사용자
                                                                          ↑
                                                                          ↓
                                                                     [반복]
```

이 사이클이 *작동하면 서비스가 시간이 갈수록 좋아짐*. 작동 안 하면 *경쟁자에게 추월당함*.

---

## 6. A/B 테스트와의 결합

새 모델을 배포할 때 *피드백 데이터로 A/B 결과 보강*:

- *명시적 만족도* 비교
- *재방문율* 비교
- *세션 길이* 비교

## 권 정리

- 명시 vs 암시 피드백 = 정확도 vs 수집 비용
- Turn vs Session-level = 세밀함 vs 마찰
- 자기 선택·생존자 편향 인식·보정
- SFT·DPO·Reward Model 데이터로 변환
- 데이터 플라이휠 = 서비스 차별화 핵심

가장 기억할 한 줄: **"좋은 AI 제품과 평범한 AI 제품의 가장 큰 차이는 모델이 아니라 데이터 플라이휠이며, 피드백 시스템이 그 시작이다."**
